## 核心原则

1. **先结构，再行文** — 不建好大纲不动笔
2. **第三人称有限视角** — POV不跳，读者代入不破
3. **章章有钩子** — 前200字抓人，后200字留人
4. **对话驱动** — 单章对话占比不低于25%
5. **爽点有节奏** — 每卷至少规划12-15个爽点，均匀分布
6. **伏笔必有收** — 埋下的每一处都要有对应回收
7. **角色有体系** — v0.0.2 每个重要角色都有完整角色卡
8. **空间有逻辑** — v0.0.2 用地图系统保证时空一致性
9. **范文参照（可选）** — 如有同类型标杆作品，做量化对比学习技法

## 起点签约核心标准

### 黄金三章
- 前500字：建立冲突 + 主角登场 + 核心卖点暗示
- 第一章：主角是谁 + 他要什么 + 阻碍是什么 + 一个爽点或钩子
- 第二章：金手指登场 + 第一次小爆发（打脸/装逼）+ 章末留钩子
- 第三章：格局升级 + 主线浮现 + 再来一次爽点 + 强钩子

### 章节规范
- 每章 2000-3000 字（推荐 2500 字）
- 短段落排版：3-5 行一段，适合手机阅读
- 对话推进剧情，减少大段旁白
- 世界观信息碎片化投喂

### 爽点密度
| 类型 | 间隔 | 说明 |
|------|------|------|
| 小爽点 | 每2000-3000字 | 一句反击、一次小胜、一个收获 |
| 中爽点 | 每3-5章 | 副本结算、阶段性胜利、关系突破 |
| 大爽点 | 每20-30章 | 大boss战、身份揭露、逆天改命 |

憋屈段不超过1-2章，压抑后必须有释放。

### 章节结构
- 开头200字内制造悬念或冲突
- 每章推进至少一件事（实力/关系/谜团）
- 结尾必须留钩子

### plan — 全书大纲

生成或修改全书大纲。

**执行步骤：**
1. 检测项目存在形式：优先 `大纲/总纲.md`，其次 `outline/master-outline.md`
2. 如已存在，读取并询问要修改哪部分
3. 如不存在，与用户对话确认：题材、核心卖点、主角设定、金手指、目标字数、世界观基调
4. 生成完整大纲，包含：
   - 一句话核心卖点
   - 主角弧光（起点→终点）
   - 金手指及其限制
   - 三卷式结构（引入→展开→收束）
   - 分卷规划（每卷主题/字数/章节/核心事件）
   - 副本列表（地点+冲突+爽点+收获）
   - 人物关系网草稿
   - 爽点节奏总表
   - 伏笔清单
5. 询问是否需要同步创建初始地图和角色卡索引

---

### character — 角色卡

创建或编辑角色卡。`$target` 为角色名。

**执行步骤：**
1. 读取 `templates/character-card.md` 获取模板
2. 如果 `角色卡/$target.md` 已存在，读取并询问要修改什么
3. 如果不存在，与用户对话确认角色定位
4. 按模板生成完整角色卡：
   - 基本信息 + 核心三要素
   - 外貌标签 + 语言风格
   - 能力/战力体系 + 成长曲线
   - 角色关系图谱（ASCII 关系图）
   - 分卷弧线追踪
   - 出场章节索引
   - 情绪/状态关键词标签
   - 关联地点（链接地图系统）
5. 更新 `角色卡/index.md` 索引
6. 提示是否更新 `角色卡/关系网.md`

---

### outline — 分卷章纲

生成分卷章纲。`$target` 为卷号。

**执行步骤：**
1. 读取全书大纲
2. 为指定卷生成详细章纲，每章包含：
   - 章节号 + 暂定标题
   - 本章目标（推进实力/关系/谜团）
   - 场景列表（地点、在场人物）
   - 事件/冲突
   - 爽点类型和位置
   - 章末钩子
   - 伏笔（埋设/回收）
   - 情绪曲线
3. 自动关联地图系统，生成 "卷内场景-地点映射表"
4. 输出爽点分布检查表

---

### write — 正文写作

撰写正文。`$target` 为章节号。

**执行步骤：**
1. 确定所属卷，读取对应章纲
2. 加载上下文：
   - 前2章正文（保持连贯）
   - 出场角色的角色卡
   - 章节涉及的地图地点卡（获取环境调色板）
   - 活跃伏笔清单
3. 起草正文，规则：
   - 第三人称有限视角，不换 POV
   - 2200-2800 字
   - 前200字钩子，后200字钩子
   - 短段落，对话占比 25%-55%
   - 至少一个小爽点
   - 用动作替代心理旁白
4. 写入 `正文/vol-X/chapter-XXX.md`
5. 输出自检报告
6. 新地点提示创建地图卡

---

### revise — 修订润色

修订润色。`$target` 为章节号或批次名（A/B/C）。

**单章修订**（target 为章节号）：按节奏/钩子/排版/对话/信息/人设/地点一致性检查修改。

**批次修订**（target = A/B/C）：
- **A批次** — 人物弧光 + 钩子 + 对话比
- **B批次** — 伏笔 + 金手指 + 精简
- **C批次** — 网文化 + 道具呼应 + 风格统一

**执行顺序：** A → 存盘 → B → 存盘 → C → 存盘 → 终审

---

### check — 一致性检查

**执行步骤：**
1. 读取所有角色卡、大纲、正文、地图地点卡、伏笔清单
2. 检查：人设一致性、伏笔追踪、设定一致性、时间线、地图一致性、空间逻辑
3. 输出问题清单，按严重程度排序

---

### foreshadowing — 伏笔管理

`$target` 可选：list / check / update。

- list：输出当前伏笔清单状态（活跃/已回收/已废弃）
- check：扫描正文，对照伏笔清单检查遗漏
- update：扫描正文更新伏笔状态

---

### map — 地图系统管理

`$target` 为地点名或操作（add / list / check）。

**地点卡创建/编辑**（target 为地点名）：
1. 读取 `templates/map-location-card.md` 获取模板
2. 生成/编辑地点详情卡：
   - 地理信息、方位关系图、路线连接
   - 剧情功能、关键事件总览
   - 出没角色（链接角色卡）
   - 势力归属变迁
   - 环境描写调色板
   - 伏笔关联
3. 更新 `地图/index.md` 索引

**list**：列出所有地点
**check**：检查正文中地点引用是否与地图卡一致

---

### map-index — 地图故事索引

**核心功能**：按地图索引查询故事内容。`$target` 为地点名或 timeline。

**按地点查事件**（target = 地点名）：
1. 读取 `地图/$target.md`
2. 扫描所有章节，提取该地点发生的所有事件
3. 输出 "地点-事件时间线"（事件 + 角色 + 伏笔 + 势力归属）

**全书时空线**（target = timeline）：
1. 按章节顺序列出所有地点的事件流转
2. 自动检测时空逻辑矛盾（角色瞬移、路线不匹配等）

## 写作风格硬性规则

- **视角**：紧贴主角视角，用主角的认知和情感过滤信息
- **节奏**：快节奏推进，删除一切不推动剧情的文字
- **对话**：每个角色有独特语言风格，对话有潜台词和冲突
- **描写**：动作 > 心理 > 环境，环境描写不超过2句
- **信息**：设定信息通过剧情自然带出，禁止旁白解释
- **张力**：每个场景都有微冲突
- **副词禁用**：用具体动作替代"非常/很/突然/缓缓"

| 指标 | 合格线 | 优秀线 | 测量方式 |
|------|--------|--------|---------|
| 单章字数 | 2000-3000 | 2200-2800 | `wc -m` |

| 前200字钩子 | 100% | 100% | 逐章检视 |
| 后200字钩子 | ≥90% | 100% | 逐章检视 |
| 伏笔回收率 | ≥80% | ≥95% | 伏笔清单统计 |
| 角色卡覆盖率 | 常驻角色100% | 阶段角色80% | 角色卡索引 |
| 地图地点覆盖 | 核心场景100% | 所有场景80% | 地图索引 |
| 地点-章节一致性 | 100% | 100% | 自动检测 |
