# 网络小说助手

## 定位

这是一套从零开始创作长篇网络小说的完整工作流。基于《挑担》（82章/25万字）实际创作经验总结，覆盖从世界观设定到最终定稿的全生命周期。

## 核心原则

1. **先结构，再行文** — 不建好大纲不动笔
2. **第三人称有限视角** — POV不跳，读者代入不破
3. **章章有钩子** — 前200字抓人，后200字留人
4. **对话驱动** — 单章对话占比不低于25%
5. **爽点有节奏** — 每卷至少规划12-15个爽点，均匀分布
6. **伏笔必有收** — 埋下的每一处都要有对应回收
7. **范文参照（可选）** — 如有同类型标杆作品，做量化对比学习技法；如无合适范文或创意足够独立，可跳过

### 总纲要素

总纲包含但不限于：
- **世界观基调**：都市/灵异/修仙/玄幻 — 确定核心氛围
- **力量体系**（如有）：等级划分、能力规则、代价限制
- **主线脉络**：主角目标、核心冲突、终局画面
- **三卷结构**：启（引入）→ 承（展开）→ 合（收束）
- **人物关系图**：核心角色 + 配角 + 反派的关联与动机

## 二、大纲设计

### 三卷式结构（推荐）

| 卷 | 功能 | 字数参考 | 节奏 |
|----|------|---------|------|
| 卷一：引入 | 建立世界观、抛出主线钩子、主角获得金手指 | 8-10万 | 慢→渐快 |
| 卷二：展开 | 冲突升级、势力扩编、伏笔密集投放 | 12-15万 | 快→极快 |
| 卷三：收束 | 所有伏笔回收、终极冲突、情感落地 | 6-8万 | 极快→慢 |

### 每卷章节规划表示例

```
| 章号 | 核心事件 | 爽点类型 | 伏笔埋设 | 伏笔回收 | 预期字数 |
|------|---------|---------|---------|---------|---------|
| 001  | 主角回村 | 身份反差 | 老宅担子 | - | 2500 |
| 002  | 二叔对话 | 信息悬念 | 狐仙传说 | - | 2500 |
```

### 章纲三要素

每章大纲明确：
1. **本章目标** — 推进哪条线？（主线/支线/人物弧光）
2. **开场钩子** — 前200字如何抓人？（冲突/悬念/强情绪）
3. **收尾钩子** — 后200字如何留人？（选择钩/危机钩/悬念钩）

## 三、范文参照（可选）

> **🤖 本模块为可选项**。除非你主动要求"参考范文分析"，助手默认跳过此步骤。
> 当你对节奏/结构不确定，或想学习同类型头部作品的技法时启用；当你创意已足够完整、或找不到合适范文时，直接跳过不影响后面的流程。

### 启用方式

在对话中说"参考范文分析"或"对标一下XXX"，助手会：
1. 询问你要参考哪部作品
2. 进行定量分析（章节长度/对话密度/钩子位置/段落节奏）
3. 进行定性分析（章首钩子/章尾钩子/爽点节奏/对话风格）
4. 输出技法转化建议

### 定量分析维度

| 维度 | 分析内容 | 工具 |
|------|---------|------|
| 章节长度 | 平均每章字数、波动范围 | `wc -m` |
| 对话密度 | 对话框占总行数比例 | `grep -c "「"` |
| 钩子位置 | 前200字钩子率、后200字钩子率 | 逐章检视 |
| 段落节奏 | 平均段落长度（短=快节奏） | `awk '{print length}'` |

### 定性分析维度

| 维度 | 分析内容 |
|------|---------|
| 章首钩子类型 | 冲突钩/悬念钩/异常钩/动作钩/对话钩 |
| 章尾钩子强度 | 弱/中/强（弱=需改进，强=可学习） |
| 爽点分布节奏 | 每几章一个爽点？最长铺垫期是几章？ |
| 对话风格 | 句式长度、潜台词、意图冲突设计 |
| 信息投放 | 什么时候告诉读者什么信息 |

### 技法转化（关键步骤）

分析范文的**目的不是模仿内容，而是学习结构**：

```
范文技法 → 你的应用

技法：范文用"日常中的微小异常"制造悬念
应用：你在自己的设定中找一个类似的异常切入点
差异：情节完全不同，但悬念机制是一样的
```

**禁止行为**：
- ❌ 抄袭情节、设定、人物关系
- ❌ 复制句式或段落
- ✅ 只学结构比例、信息节奏、情感曲线

### 详细参考

完整的范文参照方法论、实操命令、案例分析详见：
➡️ `references/范文参照.md`

## 四、写作流程

### Step 1：加载上下文

```text
读取以下文件：
- 大纲/第N卷-章节规划.md（当前卷全部规划）
- 大纲/总纲.md（世界观约束）
- 伏笔清单.md（当前阶段活跃伏笔）
- 设定集/人物卡.md（出场角色一致性检查）
- 上一章正文（保持风格/节奏连贯）
```

### Step 2：起草正文

**核心规则**：
- 第三人称有限视角，全程不换POV
- 前200字必须出现钩子（冲突/悬念/异常/强情绪）
- 后200字必须设置未闭合问题（留人继续读）
- 单章字数 2200-2800 字，稳定节奏
- 对话占比不低于25%，不多于55%
- 多用动作描写替代心理旁白
- 减少"非常/很/突然"等副词，用具体动作替代

**对话写作要点**：
- 每段对话带意图冲突（两人目标不同）
- 少用"XX说道/XX淡淡道"，用动作引导对话
- 避免超过5轮的纯对话无动作/环境穿插

### Step 3：写入伏笔清单

每完成3-5章，更新伏笔清单：
```
| 伏笔 | 埋设章节 | 类型 | 预期回收章节 | 状态 |
|------|---------|------|------------|------|
| 担子上三个结 | ch-006 | 物品 | ch-05x | 待收 |
```

### Step 4：章节自检

初稿完成后执行：
- [ ] 前200字有钩子
- [ ] 后200字有未闭合问题
- [ ] 对话比 ≥ 25%（可用 `grep -c "「"` 快速估算）
- [ ] 无 POV 跳跃
- [ ] 无 AI 痕迹句式（"他非常愤怒"→动作+生理+决策）
- [ ] 无连续三段以上相似句式
- [ ] 字数在 2200-2800 范围内

## 五、三批次修订体系

这是本技能的核心方法论，源自《挑担》创作实战。将全部修订工作分为三个批次依次执行。

### A批次 — 人物弧光 + 钩子 + 对话比

**目标**：让每个角色鲜活，每章入口出口有力，对话推动剧情。

```
检查清单：
□ 主角有清晰的情感弧线（每卷至少1次成长/转变）
□ 配角不是工具人（每人有独立动机）
□ 反派行为逻辑自洽（不降智）
□ 前200字钩子覆盖率达100%
□ 后200字钩子覆盖率达90%+
□ 单章对话比 25%-55%
□ 对话包含意图冲突（无效寒暄改写）
```

**批量执行方法**：
```bash
# 扫描对话比低的章节
grep -c "「" vol-2/*.md | sort -t: -k2 -n | head -10

# 扫描无对话框的章节
for f in vol-2/*.md; do
  count=$(grep -c "「" "$f")
  total=$(wc -l < "$f")
  ratio=$((count * 100 / total))
  if [ $ratio -lt 5 ]; then echo "$f: $ratio%"; fi
done
```

### B批次 — 伏笔 + 金手指 + 精简

**目标**：确保伏笔体系完整，金手指有成长弧线，无冗余描写。

```
检查清单：
□ 伏笔清单完整：埋设位置 / 类型 / 预期回收
□ 所有伏笔已在预期窗口内回收（或明确标记为系列文留白）
□ 金手指有使用限制和成长（不过早满级）
□ 金手指的每次升级有明确触发条件
□ 删除连续3处以上同类环境描写（月光/秋风等）
□ 删除连续200字以上的纯解释段落
□ 全文同一物象用词一致（不混用"扁担"/"担杖"）
□ 无过度重复的拟声词/副词（叮叮当当/突然/缓缓）
```

**高效批量扫描方法**：
```bash
# 扫描高频重复词
grep -oP '[^一-龥]*[一-龥][一-龥][一-龥]?[一-龥]?[^一-龥]*' *.md | sort | uniq -c | sort -rn | head -30

# 扫描重复的拟声词/环境词
grep -n "叮叮当当\|月光\|秋风\|缓缓\|突然" *.md | cut -d: -f1 | sort | uniq -c | sort -rn
```

### C批次 — 网文化 + 道具呼应 + 风格统一

**目标**：增强网文阅读爽感，强化道具/场景的叙事功能，保证全文风格一致。

```
检查清单：
□ 章末有悬疑钩/选择钩/危机钩之一（非平铺直叙结尾）
□ 爽点密度：每卷至少3个密集爽点区
□ 重要道具在关键场景重复出场（建立情感关联）
□ 角色口头禅/标志性动作保持连贯
□ 无"总而言之/可以说/值得注意的是"等元叙述
□ 战斗/冲突场面有节奏变化（起→承→转→合）
□ 无 AI 万能句式（"眸中闪过一丝"/"缓缓说道"/"心中暗道"）
□ 同一概念在全文用同一词汇（不随意替换同义词）
```

## 六、伏笔管理

### 伏笔类型

| 类型 | 定义 | 示例 | 回收周期 |
|------|------|------|---------|
| 物品伏笔 | 关键道具 | 担子上的三个结 | 中（3-5卷） |
| 人物伏笔 | 未揭示的身份/动机 | 赵家地基的镇物 | 中 |
| 设定伏笔 | 世界规则暗示 | "往西走" | 长（终卷） |
| 事件伏笔 | 前因后果 | 狐仙被镇压的原因 | 短（1-2卷） |
| 对话伏笔 | 看似随意的对话 | "你走路的声音重了" | 短-中 |

### 伏笔清单模板

```markdown
# 伏笔清单

## 活跃中
| ID | 伏笔 | 埋设章 | 类型 | 载体 | 预期回收 | 备注 |
|----|------|--------|------|------|---------|------|
| F01 | 担子三个结 | ch-006 | 物品 | 扁担绳扣 | vol-2后半 | 内藏镇符 |
| F02 | 向西走 | ch-009 | 设定 | 胡秀兰对话 | vol-3 | 最终目的地 |

## 已回收
| ID | 伏笔 | 回收章 | 回收方式 |
|----|------|--------|---------|
| F03 | 石头会发热 | ch-010 | 确认有灵性 |

## 已废弃
（记录废弃原因，避免后续误追）
```

## 七、质量基准

### 数据指标

| 指标 | 合格线 | 优秀线 | 测量方式 |
|------|--------|--------|---------|
| 单章字数 | 2000-3000 | 2200-2800 | `wc -m` |
| 对话占比 | ≥25% | 30-45% | `grep -c "「"` / 总行数 |
| 前200字钩子 | 100% | 100% | 逐章人工检视 |
| 后200字钩子 | ≥90% | 100% | 逐章人工检视 |
| 伏笔回收率 | ≥80% | ≥95% | 伏笔清单统计 |
| 重复描写的章节 | ≤20% | ≤10% | `grep -c` 高频词 |
| 连续解释段长度 | ≤200字 | ≤150字 | 目检 |
| 第三人称一致性 | 100% | 100% | `grep "我"` 检查对话外使用 |

### 范文参照（可选项）

当你需要学习同类型标杆作品的技法时，启用范文参照模块（详见第三章）：

```bash
# 启用范文参照
用户说："参考一下《大王饶命》的写法"

# 助手执行定量分析
grep -c "「" 范文/第*.md | awk -F: '{sum+=$2; count++} END {print "平均每章对话行:", sum/count}'
wc -m 范文/第*.md | tail -1

# 输出分析报告 + 技法转化建议
```

**不启用时**：跳过此步骤，不影响后续流程。基于自身大纲直接进入写作。

## 八、常见陷阱与对策

| 陷阱 | 表现 | 对策 |
|------|------|------|
| POV漂移 | 突然切换到配角视角 | 每次换场景检查主语是不是主角 |
| 工具人配角 | 配角只在需要时出现 | 给每个常驻配角写独立动机卡 |
| 伏笔遗忘 | 前半埋的线后半断了 | 每日更新伏笔清单 |
| 描写冗余 | 每章都写月光/秋风 | B批次集中扫描+删除 |
| 对话过少 | 大段叙述无对话 | A批次针对性补充冲突对话 |
| 节奏失衡 | 连续多章无爽点 | 参考爽点分布表做加插 |

## 九、修订批次执行顺序

无论修订规模大小，严格按此顺序执行：

```
A批次（人物/钩子/对话）→ 存盘 → B批次（伏笔/金手指/精简）
→ 存盘 → C批次（网文化/道具/风格）→ 存盘 → 终审
```

**禁止跨批次操作**：A批次未完成的章节不进B批次，B批次未完成的章节不进C批次。每一批次可能对上一批次的修改产生连锁影响（如B批次精简可能删掉A批次刚加的对话），需要逐批确认。
