# 地图系统使用指南

## 为什么需要地图系统

长篇网文（尤其是玄幻/修仙/灵异题材）涉及大量地理空间信息。传统创作工具只有"时间线"维度，缺少"空间线"维度。地图系统解决了以下痛点：

| 痛点 | 地图系统解法 |
|------|-------------|
| 角色瞬间移动/穿越时空不自洽 | 路线连接 + 旅行时间记录 |
| 写了半天发现场景天气/氛围矛盾 | 环境调色板统一管理 |
| 忘了某个地点之前发生过什么 | 地点-事件索引查询 |
| 势力版图变化记不清 | 势力归属变迁追踪 |
| 读者问"从A到B要多久"答不上来 | 路线连接表提供参考 |

## 地图层级结构

建议按以下层级组织地图文件：

```
地图/
├── index.md                    # 世界地图总索引
├── 大陆-北玄大陆.md            # 大陆级
├── 国家-大炎帝国.md            # 国家级
├── 城市-帝都.md                # 城市级
├── 宗门-青云宗.md              # 具体地点级
├── 副本-落霞秘境.md            # 副本/秘境级
└── 势力分布.md                 # 势力版图变迁追踪
```

### 层级关系原则

1. **上层包含下层** — 大陆包含国家，国家包含城市
2. **每个地点只属于一个上级** — 避免归属混乱
3. **重要度决定粒度** — 路边小店只需要一行备注，宗门驻地需要完整地点卡
4. **不必一次建全** — 在写作过程中按需创建，写到哪建到哪

## 如何创建地图地点

### 场景1：大纲阶段批量创建

执行 `plan` 或 `outline` 时，技能会自动询问是否创建地图地点。批量创建流程：

1. 列出大纲中所有提到的地点
2. 按重要度分类：
   - **核心场景**（如主角宗门、帝都、最终决战地）→ 完整地点卡
   - **次要场景**（如路过的小镇）→ 基础信息即可
   - **一次性场景** → 在章纲中备注，不建卡
3. 按层级关系组织
4. 生成 `地图/index.md` 总览

### 场景2：写作过程中按需创建

执行 `write` 时，如章纲涉及新地点且较为重要，技能会提示：

```
检测到本章涉及新地点「青云宗后山」。
是否创建该地点的地图卡？(是/否/简单备注)
```

选"简单备注"则只在章纲中留下记录，后续需要时再补卡。

### 场景3：手动补卡

```
/map 青云宗    # 创建或编辑青云宗地图卡
/map list      # 查看所有已建地图卡
/map check     # 检查正文中所有地点引用是否与地图卡一致
```

## 势力分布追踪

`地图/势力分布.md` 是一个特殊的跟踪文件，按时间线记录各势力的版图变化：

```markdown
# 势力分布变迁

## 初始状态（ch-000）
| 势力 | 控制区域 | 备注 |
|------|---------|------|
| 大炎帝国 | 北玄大陆中部平原 + 帝都 | 正统皇权 |
| 青云宗 | 青云山脉一带 | 正道领袖 |
| 魔教 | 黑森林以南（暗中） | 隐藏状态 |

## 第一次洗牌（ch-028~ch-030 魔教攻山后）
| 势力 | 控制区域 | 变化 |
|------|---------|------|
| 魔教 | 增加青云山脉 | 从隐藏转为半公开 |

## 第二次洗牌（ch-045 主角登基后）
| 势力 | 控制区域 | 变化 |
|------|---------|------|
| 主角阵营 | 帝都 + 青云宗 + 南部三城 | 势力扩张 |
```

## 地图中的伏笔管理

地图系统与伏笔系统深度集成：

1. **地点本身可以成为伏笔** — "主角在青云宗后山发现了一个神秘洞穴"
   → 在地点卡中标记 `伏笔关联` 字段，链接到伏笔清单

2. **势力归属变化可能是伏笔回收** — "原来魔教一直在黑森林养精蓄锐"
   → 势力分布变迁中记录"黑森林早在第X章就暗示过"

3. **路线/距离伏笔** — "从A到B需要3天，但主角1天就到了"
   → 可能暗示主角实力飞跃 / 有人暗中相助 / 地图本身有问题

## 写作时的地图引用

在正文写作中，地图系统提供以下直接帮助：

1. **环境描写一致性**
   ```markdown
   写到「青云宗」时，自动加载该地的环境调色板：
   - 视觉：云雾缭绕、青石台阶、松柏成林
   - 听觉：晨钟、松涛、弟子练功声
   - 嗅觉：松香、晨露、香火味
   避免：写到一半突然出现本来没有的"瀑布声"
   ```

2. **角色位置追踪**
   ```markdown
   主角当前在「帝都」，可交互角色列表：
   - 公主（在皇宫）
   - 老师（在国子监）
   - 对手（在城北演武场）
   不可交互：大师兄（在3000里外的青云宗）
   ```

3. **旅行时间参考**
   ```markdown
   从「帝都」到「青云宗」的已知路线：
   - 骑马：10天
   - 御剑：3天
   - 传送阵：瞬间（但已损坏）
   如果让主角3天骑马赶到，需要合理解释
   ```

## 常见问题

| 问题 | 对策 |
|------|------|
| 创建太多地点卡管理不过来 | 只为核心场景建卡，次要场景在章纲中备注即可 |
| 地点之间的路线不确定 | 留空，写到了再补充；或者用"约X天路程"模糊处理 |
| 势力归属变化太频繁 | 只在有重大转折时更新势力分布，小范围拉锯战不必记录 |
| 正文中的地点和地图卡对不上 | 执行 `map check` 自动扫描不一致处 |
| 前期没建地图系统，现在想补 | 执行 `map-index timeline` 扫描正文中所有地点后批量创建 |
