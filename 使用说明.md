# 网络小说助手 v0.0.2 使用说明

## 概述

本技能是专为长篇网络小说创作设计的 Claude Code 助手，适用于起点中文网、番茄小说、七猫等平台风格的网文创作。覆盖从 0 到 1 的全流程：设定 → 大纲 → 角色卡系统 → 地图索引 → 写作 → 三批次修订 → 定稿。

**v0.0.2 新增**：角色卡系统和地图故事索引系统。

---

## 适用场景

- **新书启动**：已有故事的灵感/设定，需要系统的创作流程
- **连载写作**：需要稳定质量标准的日更/周更写作
- **全文修订**：已完成初稿，需要系统化审校和润色
- **复杂多角色故事**：需要管理大量角色关系和活动轨迹
- **时空逻辑要求高的题材**：玄幻/修仙/探险类，需要保证地点一致性

## 快速开始

### 第一步：初始化项目

```text
/网络小说助手 plan
```

技能会创建如下目录结构（v0.0.2 新增 `角色卡/` 和 `地图/` 目录）：

```
项目目录/
├── 大纲/
│   ├── 总纲.md
│   └── 第1卷-章节规划.md
├── 正文/
│   └── vol-1/
├── 角色卡/              ← v0.0.2 新增
│   ├── index.md
│   └── 关系网.md
├── 地图/                ← v0.0.2 新增
│   ├── index.md
│   └── 势力分布.md
├── 伏笔清单.md
├── 修订记录.md
└── 审查日志.md
```

### 第二步：设计总纲

与助手对话确定以下内容：
1. 世界观/题材（都市灵异/玄幻修仙/规则怪谈等）
2. 主角设定（身份、性格、目标、缺陷）
3. 核心冲突（主线悬念）
4. 力量体系（如有）
5. 三卷结构（启→承→合）
6. 金手指类型（如有）

### 第三步：创建角色卡（v0.0.2 新增）

为每个重要角色创建角色卡：

```text
/网络小说助手 character 主角名
/网络小说助手 character 配角名
```

角色卡包含：核心三要素、关系图谱、能力成长曲线、出场章节索引、情绪标签等。

当有多个角色后，可维护关系网：

```text
/网络小说助手 character 关系网    # 更新全局关系图谱
```

### 第四步：创建地图地点（v0.0.2 新增）

为关键场景创建地图地点卡：

```text
/网络小说助手 map 青云宗
/网络小说助手 map 帝都
/网络小说助手 map list              # 查看所有地点
```

地点卡包含：方位关系、环境调色板、势力归属变迁、伏笔关联等。

### 第五步：分卷规划

```text
/网络小说助手 outline 1
```

每卷大纲包含：章节清单、爽点分布图、伏笔投放计划、人物弧光变化。

### 第六步（可选）：范文参照

如果你希望学习同类型优秀作品的技法，此时可以要求助手进行范文分析：

```text
"参考一下《大王饶命》的写作手法"
```

### 第七步：逐章写作

```text
/网络小说助手 write 1
/网络小说助手 write 2
```

写作流程：加载章纲 + 角色卡 + 地图卡 + 伏笔 → 起草 → 自检 → 更新。

### 第八步：地图索引查询（v0.0.2 新增）

写作过程中或完成后，按地点检索故事内容：

```text
/网络小说助手 map-index 青云宗    # 查询该地点所有事件时间线
/网络小说助手 map-index timeline   # 查看全书时空线，检测时空矛盾
```

### 第九步：批量修订

```text
/网络小说助手 revise A    # A批次：人物弧光+钩子+对话比
/网络小说助手 revise B    # B批次：伏笔+金手指+精简
/网络小说助手 revise C    # C批次：网文化+道具呼应+风格统一
```

### 第十步：一致性检查

```text
/网络小说助手 check       # 全本一致性（含地图一致性）
/网络小说助手 report      # 项目进度报告
```

## 修订批次详解

### A批次：人物弧光+钩子+对话比

**检查清单**：
- 主角有清晰的情感弧线（每卷至少1次成长/转变）
- 配角不是工具人（每人有独立动机）
- 反派行为逻辑自洽（不降智）
- 前200字钩子覆盖率达100%
- 后200字钩子覆盖率达90%+
- 单章对话比 25%-55%
- 对话包含意图冲突

**批量扫描**：
```bash
grep -c "「" 正文/vol-*/*.md | sort -t: -k2 -n | head -10
```

### B批次：伏笔+金手指+精简

**检查清单**：
- 伏笔清单完整
- 所有伏笔已在预期窗口内回收
- 金手指有使用限制和成长
- 删除连续3处以上同类环境描写
- 删除连续200字以上的纯解释段落
- 全文同一物象用词一致

**批量扫描**：
```bash
# 扫描高频环境词
grep -n "月光\|秋风\|夕阳\|清晨" vol-2/*.md | cut -d: -f1 | sort | uniq -c | sort -rn
# 找超长段落
awk 'length > 200' *.md
```

### C批次：网文化+道具呼应+风格统一

**检查清单**：
- 章末有悬疑钩/选择钩/危机钩之一
- 爽点密度达标
- 重要道具在关键场景重复出场
- 无 AI 万能句式
- 角色口头禅/标志性动作保持连贯
- 同一概念在全文用同一词汇

## 角色卡系统简介

v0.0.2 的角色卡比传统人设卡更强大：

| 功能 | 说明 |
|------|------|
| **关系图谱** | ASCII 关系图可视化角色间联系 |
| **分卷弧线** | 逐卷追踪角色状态变化 |
| **能力成长曲线** | 战力/技能变化追踪 |
| **出场章节索引** | 自动记录角色出场章节 |
| **情绪标签** | 关键词标签化角色状态 |
| **关联地点** | 链接到地图系统 |

创建角色卡后，关系由 `角色卡/关系网.md` 统一管理。

## 地图系统简介

v0.0.2 新增的地图系统从空间维度管理故事：

| 功能 | 说明 |
|------|------|
| **地点详情卡** | 地理信息、环境调色板、势力和伏笔关联 |
| **方位关系图** | ASCII 地图可视化地点间位置关系 |
| **势力归属变迁** | 追踪各势力控制区域变化 |
| **事件时间线** | 按地点检索所有相关事件 |
| **全书时空线** | 自动检测时空逻辑矛盾 |
| **路线连接** | 记录地点间距离和旅行时间 |

详细用法见 `guidelines/map-system-guide.md`。

## 最佳实践

1. **先建角色卡再写作** — 重要角色提前建卡，避免后期人设崩塌
2. **边写边建地图** — 写到新场景时顺手建地点卡，不必一次建全
3. **定期 map-index** — 每写10章跑一次 `map-index timeline` 检查时空逻辑
4. **保持一致**：每天写相同字数，维持节奏感
5. **频繁自检**：每写5章做一次小规模检查（对话比/伏笔状态）
6. **不要恋战**：某一章卡住时写个占位符继续推进，回头再修
7. **批量优于逐章**：全书写完再修订，比每章写完美效率高10倍

## 故障排除

| 问题 | 原因 | 解决 |
|------|------|------|
| 上下文溢出 | 章节太多 | 用 `Agent` 工具做独立章节修订 |
| 钩子重复 | 模式固定 | 参考钩子类型表轮换 |
| 节奏拖沓 | 爽点间隔太长 | 插入小型冲突/反转 |
| 角色扁平 | 缺乏弧光设计 | 用 `character` 命令补全角色弧线 |
| 空间矛盾 | 角色位置跳跃 | 用 `map-index timeline` 检测并补过渡章节 |
| 对话生硬 | 缺乏意图冲突 | 确保每次对话双方目标不同 |
