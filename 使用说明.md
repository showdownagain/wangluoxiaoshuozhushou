# 网络小说助手 使用说明

## 概述

本技能是专为长篇网络小说创作设计的Claude Code助手，适用于起点中文网、番茄小说、七猫等平台风格的网文创作。覆盖从0到1的全流程：设定→大纲→写作→修订→定稿。

## 适用场景

- **新书启动**：已有故事的灵感/设定，需要系统的创作流程
- **连载写作**：需要稳定质量标准的日更/周更写作
- **全文修订**：已完成初稿，需要系统化审校和润色
- **风格转换**：需要将非网文风格（如知乎体、第一人称）转换为网文风格
- **技法学习（可选）**：想要通过分析同类型标杆作品，学习优秀写作技法

## 快速开始

### 第一步：初始化项目

```text
/claude-code init-webnovel
```

技能会创建如下结构：
```
项目目录/
├── 大纲/
│   ├── 总纲.md
│   └── 第1卷-章节规划.md
├── 正文/
│   └── vol-1/
├── 设定集/
├── 伏笔清单.md
├── 修订记录.md
└── 审查日志.md
```

### 第二步：设计总纲

与助手对话确定以下内容：
1. 世界观/题材（都市灵异/玄幻修仙/规则怪谈等）
2. 主角设定（身份、性格、目标、缺陷）
3. 核心冲突（主线悬念）
4. 力量体系（如有）
5. 三卷结构（启→承→合）
6. 金手指类型（如有）

### 第三步：分卷规划

每卷大纲包含：
- 本卷核心冲突与目标
- 章节清单（每章100字简介）
- 爽点分布图（每卷12-15个爽点）
- 伏笔投放计划
- 人物弧光变化

### 第四步（可选）：范文参照

如果你希望学习同类型优秀作品的技法，此时可以要求助手进行范文分析：

```text
"参考一下《大王饶命》的写作手法"
"对标一下这本小说"
```

助手会：
1. 定量分析（章节长度、对话密度、段落节奏）
2. 定性分析（钩子类型、爽点节奏、对话风格）
3. 输出技法转化建议（只学结构，不抄内容）

**跳过此步骤**：如果你的创意足够独立，或没有合适的范文，直接进入下一步即可。

### 第五步：逐章写作

写作流程：
1. 助手加载当前章纲、活跃伏笔、人物设定
2. 起草正文（遵循第三人称有限视角）
3. 章节自检（钩子/对话比/字数/POV）
4. 更新伏笔清单
5. 继续下一章

### 第六步：批量修订

全文完成后执行三批次修订：
1. **A批次**：人物弧光→章钩子→对话比
2. **B批次**：伏笔回收→金手指弧线→描写精简
3. **C批次**：网文化→道具呼应→风格统一

## 常用命令

| 命令 | 功能 |
|------|------|
| `/write-chapter N` | 写第N章正文 |
| `/review-chapter N` | 审查第N章（加载检查清单） |
| `/batch-revision A` | 执行A批次修订 |
| `/batch-revision B` | 执行B批次修订 |
| `/batch-revision C` | 执行C批次修订 |
| `/foreshadowing-list` | 输出当前伏笔清单状态 |
| `/check-hooks N` | 检查第N章前后200字钩子 |
| `/dialogue-ratio` | 统计全文对话占比 |
| `/reference-analysis` | 范文参照分析（对标同类作品） |
| `/reference-skip` | 跳过范文参照（使用自主创意） |

## 修订批次详解

### A批次：人物弧光+钩子+对话比

**耗时估计**：82章约需2-3轮对话
**核心方法**：
- 逐章检查前200字是否有钩子，无则加冲突/悬念/异常
- 逐章检查后200字是否有未闭合问题，无则补选择钩/危机钩
- 统计对话占比低的章节，针对性插入冲突对话
- 检查配角是否有独立动机，工具人配角补充背景

**常见操作**：
```
# 在章尾追加悬疑钩
在现有尾段后增加一句环境/状态异常描写，暗示下一章的危机

# 增加对话比
找到大段叙述段落，插入一个持有不同意见的对话角色
```

### B批次：伏笔+金手指+精简

**耗时估计**：82章约需1-2轮对话
**核心方法**：
- 建立或更新完整伏笔清单
- 扫描遗漏的伏笔回收点
- 检查金手指是否有成长的限制和代价
- 删除连续3次以上的同类环境描写
- 压缩200字以上的纯解释段落

**高效扫描命令**：
```bash
# 扫描高频环境词
grep -n "月光\|秋风\|夕阳\|清晨" vol-2/*.md | cut -d: -f1 | sort | uniq -c | sort -rn

# 检查重复拟声词
grep -n "叮叮当当\|嗡嗡\|嘎吱" *.md | wc -l

# 找超长段落
awk 'length > 200' *.md
```

### C批次：网文化+道具呼应+风格统一

**耗时估计**：82章约需1轮对话
**核心方法**：
- 增强爽点密度（每个高潮区集中爆发2-3个爽点）
- 重要道具在关键场景重复出场（建立情感关联）
- 去除"总而言之/可以说/值得注意的是"等元叙述
- 去除AI万能句式（眸中闪过一丝/缓缓说道/心中暗道）
- 确保全文同概念用词一致

## 与竞品对比

| 维度 | 本技能 | 通用AI写作 |
|------|--------|-----------|
| 方法论来源 | 《挑担》82章实战经验 | 通用写作知识 |
| 修订体系 | A/B/C三批次闭环 | 单次审查 |
| 伏笔管理 | 全生命周期跟踪 | 无 |
| 质量控制 | 量化指标+自动化扫描 | 主观判断 |
| 章节结构 | 前200/后200钩子标准 | 无明确规定 |
| 对话比 | 25%-55%量化标准 | 无 |

## 最佳实践

1. **保持一致**：每天写相同字数，维持节奏感
2. **频繁自检**：每写5章做一次小规模检查（对话比/伏笔状态）
3. **不要恋战**：某一章卡住时写个占位符继续推进，回头再修
4. **批量优于逐章**：全书写完再修订，比每章写完美效率高10倍
5. **善用命令行扫描**：用grep/awk做批量检查远比人工翻阅高效
6. **范文参照按需启用**：卡文或对节奏没把握时，用对标分析找感觉；自信时大胆跳过大纲直接推进
7. **参考对标**：定期与同类型头部作品做数据对比

## 故障排除

| 问题 | 原因 | 解决 |
|------|------|------|
| 上下文溢出 | 章节太多 | 用Agent工具做独立章节修订 |
| 钩子重复 | 模式固定 | 参考钩子类型表轮换 |
| 节奏拖沓 | 爽点间隔太长 | 插入小型冲突/反转 |
| 角色扁平 | 缺乏弧光设计 | 写角色独白/受挫/成长段落 |
| 对话生硬 | 缺乏意图冲突 | 确保每次对话双方目标不同 |
