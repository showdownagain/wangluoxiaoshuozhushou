# 三批次修订体系 — 详细参考

## 批次总览

```
A批次（人物/钩子/对话）→ B批次（伏笔/金手指/精简）→ C批次（网文化/道具/风格）
```

---

## A批次：人物弧光 + 钩子 + 对话比

### 人物弧光检查

**每章提问**：
- 本章中主角有变化吗？（哪怕很小——做决定、学到东西、改变看法）
- 配角有独立于主角的动机吗？

**修改技法**：
1. **配角工具人化修复**：在某段对话中插入配角的个人动机。"我帮你不是因为好心，是因为——"
2. **弧光缺失修复**：在章尾加一句内心对比。"以前的他，大概不会这么做。"

### 钩子类型参考

| 钩子类型 | 定义 | 示例（前200字） | 示例（后200字） |
|---------|------|----------------|----------------|
| 冲突钩 | 两个人目标对立 | "这房子你不能动。" | "他不打算退让。" |
| 悬念钩 | 一个未解答的问题 | 门上贴了三根鸡毛 | 但桌子上多了一碗水 |
| 异常钩 | 一件事违背常理 | 下山走了三圈回到同一棵树 | 石头在口袋里自己动了一下 |
| 强情绪钩 | 角色处于激烈情绪中 | 他握着电话的手在抖 | 她没哭，也没说话 |
| 动作钩 | 正在进行的高张力事件 | 他挑着担子往后山跑 | 身后的门，正在自己关上 |
| 对话钩 | 对话揭示关键信息 | "你爷死前留了一句话" | "小心赵家的人" |

### 对话比优化

**检查方法**：
```bash
# 统计每章对话行比
for f in vol-2/*.md; do
  dialogue=$(grep -c "「" "$f")
  total=$(grep -c "[。！？]" "$f")
  ratio=$((dialogue * 100 / total))
  if [ $ratio -lt 20 ]; then echo "低: $f ($ratio%)"; fi
  if [ $ratio -gt 55 ]; then echo "高: $f ($ratio%)"; fi
done
```

**对话添加原则**：
- 添加对话时必须包含意图冲突（双方对事情有不同看法）
- 避免"信息问答式"对话（A问B答，纯信息传递）
- 对话中带动作描写（在说话的同时做别的事）
- 参考格式：`动作引导 + 对话 + 反应`

**示例**：
```
❌ 原版（纯叙述）：
陈勉检查了担子。里面的东西都在。他准备出发了。

✅ 修改后（对话驱动）：
"真要挑这个走？"阿彪蹲在门槛上，叼着烟，"你不怕别人看见说你神经病？"
陈勉拍了拍筐沿："看见了又怎样，我又没犯法。"
"你是不怕，但你老婆呢？"
陈勉的手停了一下。
```

## B批次：伏笔 + 金手指 + 精简

### 伏笔回收扫描

通过以下信号判断伏笔是否遗漏：
1. **人物**：某个角色消失超过10章（除非是特意留白）
2. **物品**：关键道具被提到后30章内没再出现
3. **设定**：提到某个规则/限制后没有被触发或验证
4. **对话**：某人说"以后再说"后没有"以后"的章节

**检查方法**：打开伏笔清单，标记所有"活跃中"条目，逐条追踪。

### 金手指弧线检查

| 阶段 | 功能 | 字数范围 | 示例 |
|------|------|---------|------|
| 初期（1-20章） | 发现能力、摸索规则 | 前8万 | 石头会发热、能指路 |
| 中期（21-60章） | 能力升级、代价显现 | 8-18万 | 石头能镇邪、使用后虚弱 |
| 后期（61-82章） | 能力成熟、最终测试 | 18-25万 | 完全掌控、终极一战 |

**修改方法**：
- 如果金手指中期已经无敌→加使用代价（生命力、记忆、重要物品）
- 如果金手指成长无触发条件→加瓶颈（需要特定条件才能突破）
- 如果金手指在结尾消失→确保有充分的"为什么失去/放弃"铺垫

### 描写精简

**高频冗余词**（见到就删或替换）：
```
月光、秋风、夕阳、清晨 → 保留最好的2-3处，其余删除
叮叮当当、嗡嗡、嘎吱 → 每个拟声词在全文中出现的总次数 ≤ 5
突然、忽然、猛地 → 通过语序自然表达紧迫感
非常、很、极其 → 用具体细节替代（"非常愤怒"→"砸了手中的杯子"）
缓缓、慢慢、渐渐 → 减少使用频次，通过动作本身传达节奏
```

**压缩解释段**：
- 连续200字以上无对话无动作的段落 → 打散为2-3段，中间插入动作或对话
- 背景信息 → 分散在3-5处慢慢透露，不要一次性灌给读者

## C批次：网文化 + 道具呼应 + 风格统一

### 章末钩子强化

**标准格式**：
```
[最后的动作/对话] + [环境异常/未闭合的信息] + [暗示下一章的方向]
```

**示例**：
```
他推开门。
屋里没有人。
但桌上的茶水——还是温的。
```

### 爽点密度控制

每卷爽点分配参考（以3卷/25万字计）：

| 卷 | 字数 | 大爽点 | 小爽点 | 密度 |
|----|------|--------|--------|------|
| 卷一 | 8万 | 3-4个 | 8-10个 | 每1.5章1个小爽点 |
| 卷二 | 12万 | 4-5个 | 12-15个 | 每章1个小爽点 |
| 卷三 | 5万 | 2-3个 | 5-6个 | 每章1个爽点 |

**爽点类型建议**：
- 信息差爽点（读者知道但角色不知道 / 角色知道但读者不知道）
- 扮猪吃虎（表面弱，实际强）
- 反转（形势逆转/身份揭晓/真相大白）
- 释放（长期压抑后的爆发/对抗/胜利）

### 道具呼应

让以下类型物品贯穿全文：

**示例**（来自《挑担》）：
```
石头：ch-010出现→ch-015发热→ch-047主动引导→ch-065对话→ch-079交流
       弧线：死物→温热→有意识→会回应→有记忆→有情感

担子：ch-001出现→ch-010夜行→ch-043调整→ch-070自动适应→ch-082放下
       弧线：爷爷遗物→主角工具→成长伙伴→使命象征→传承放下
```

**检查方法**：
1. 列出全文中出现超过3次的所有物品/地点
2. 确认每一次出场都在推动某一弧线
3. 删除仅用来"凑字数"的道具出场

### AI句式替换表

| AI万能句式 | 替代方案 |
|-----------|---------|
| 他眸中闪过一丝X | 他[具体动作]了一下 |
| XX缓缓说道 | [动作]——"..." |
| XX心中暗道 | 直接内心句（不加引导词） |
| 总而言之/可以说 | 直接抛出结论 |
| 值得注意的是 | 直接写值得注意的内容 |
| 他非常O | 动作+生理反应+决策（三段式） |
| 突然/忽然 | 通过语序制造意外感 |
| 他深吸一口气 | 减少，保留关键场景即可 |
