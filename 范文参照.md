# 范文参照（可选）

> 本模块为**可选项**。当你有同类型优秀作品可以参考时，用它来学习技法；当你对自己的创意足够自信、或找不到合适的范文时，完全可以跳过。

---

## 一、范文选择指南

### 选文标准

| 维度 | 推荐 | 不推荐 |
|------|------|--------|
| 题材 | 同类型（灵异对灵异、修仙对修仙） | 跨类型对比 |
| 成绩 | 经过市场验证的头部作品 | 冷门/未完结/成绩不明 |
| 时效 | 近3年内 | 10年以上的老作品（读者审美已变） |
| 风格 | 与你的目标平台一致（起点/番茄/七猫） | 风格差异过大 |
| 数量 | 精选1-3部 | 超过5部（分析不过来的等于没分析） |

### 推荐范文组合（灵异题材示例）

- **主参考**：《大王饶命》（起点都市异能，对话/幽默标杆）
- **副参考**：《盗墓笔记》（氛围塑造、悬念设计）
- **风格参考**：《我在精神病院学斩神》（现代灵异节奏）

> 注意：不同题材需要不同的范文组合。写都市找都市参考，写修仙找修仙参考。

## 二、定量分析（数据对标）

这是我们在《挑担》创作中用过的有效方法——用数据说话，而不是"感觉"。

### 2.1 章节长度对比

```bash
# 统计范文每章字数
for f in 范文/vol-1/*.md; do
  chars=$(wc -m < "$f")
  echo "$(basename "$f"): $chars 字"
done | awk -F: '{sum+=$2; count++} END {print "平均: " int(sum/count) " 字/章"}'

# 与本作品对比
for f in 正文/vol-1/*.md; do
  chars=$(wc -m < "$f")
  echo "$(basename "$f"): $chars 字"
done | awk -F: '{sum+=$2; count++} END {print "本作品平均: " int(sum/count) " 字/章"}'
```

### 2.2 对话密度对比

```bash
# 范文对话密度
echo "=== 范文 ==="
for f in 范文/vol-1/*.md; do
  dialogue=$(grep -c "「" "$f")
  total=$(wc -l < "$f")
  ratio=$(awk "BEGIN {printf \"%.0f\", $dialogue * 100 / $total}")
  echo "$(basename "$f"): ${ratio}%"
done

# 本作品对话密度
echo "=== 本作品 ==="
for f in 正文/vol-1/*.md; do
  dialogue=$(grep -c "「" "$f")
  total=$(wc -l < "$f")
  ratio=$(awk "BEGIN {printf \"%.0f\", $dialogue * 100 / $total}")
  echo "$(basename "$f"): ${ratio}%"
done
```

### 2.3 钩子位置分析

```bash
# 分析范文前200字是否有对话（快速估算开场活跃度）
for f in 范文/vol-1/*.md; do
  has_dialogue=$(head -c 200 "$f" | grep -c "「")
  echo "$(basename "$f"): 开场有对话框=$([ $has_dialogue -gt 0 ] && echo '是' || echo '否')"
done
```

### 2.4 段落节奏对比

```bash
# 范文平均段落长度（短 ≈ 节奏快）
awk '!/^$/' 范文/vol-1/*.md | awk '{len=length} len>0 {total+=len; count++} END {print "范文平均段落长度: " int(total/count) " 字"}'

# 本作品平均段落长度
awk '!/^$/' 正文/vol-1/*.md | awk '{len=length} len>0 {total+=len; count++} END {print "本作品平均段落长度: " int(total/count) " 字"}'
```

## 三、定性分析（技法拆解）

### 3.1 章首钩子分析

选取范文前10章，逐章拆解开场方式：

```
范文第1章开场分析：
┌────────────────────────────────────────────────────┐
│ 前50字：环境快描 + 主角出场动作                      │
│ 前100字：出现异常事件（打破日常）                     │
│ 前200字：冲突/悬念建立，读者被钩住                    │
│ 手法：用"反常"制造悬念，不解释原因                    │
└────────────────────────────────────────────────────┘
```

**分析维度**：
- 开场类型（环境引入/对话切入/事件爆发/内心独白）
- 异常出现的位置（第几句？）
- 读者"想知道接下来"的驱动力是什么

### 3.2 章尾钩子分析

```
范文第1章结尾分析：
┌────────────────────────────────────────────────────┐
│ 最后100字：一个未完成的动作 + 一个未解答的问题        │
│ 最后一句：环境或状态异常，暗示下一章方向              │
│ 钩子类型：悬念钩（"那扇门自己开了"）                 │
│ 强度：强——读者必须点开下一章                        │
└────────────────────────────────────────────────────┘
```

**钩子强度评级**：
| 强度 | 标准 | 示例 |
|------|------|------|
| 弱 | 章尾平淡收束，无未闭合问题 | "他关灯睡了" |
| 中 | 有疑问但紧迫感不强 | "明天还有一件事要处理" |
| 强 | 读者必须知道接下来发生了什么 | "身后的门慢慢打开了" |

### 3.3 爽点节奏分析

统计范文每卷的爽点分布：

```
范文 卷X爽点分析：
第1章  - 微爽点（信息差：主角知道别人不知道的事）
第2章  - 中爽点（扮猪吃虎：小试牛刀）
第3-4章- 铺垫期（无大爽点，埋线）
第5章  - 中爽点（打脸权威）
第7章  - 大爽点（击败第一个真正对手）
...
```

**分析结论**：
- 范文的爽点节奏是"每2-3章一个小高潮，每5章左右一个中高潮"
- 最长铺垫期不超过4章
- 大爽点分布在前/中/后三段的黄金分割点

### 3.4 对话风格分析

```
范文对话特点：
□ 短句为主（平均每句8-15字）
□ 刻意不对称（一方话多、一方话少）
□ 带潜台词（表面说A，实际在说B）
□ 有胜负（每次对话一方达成目的、另一方没达成）
□ 用动作/环境穿插打断对话（不连续超过5轮）
```

## 四、技法转化（从学习到应用）

### 禁止做的事
- ❌ 抄袭情节/设定/人物关系
- ❌ 复制句式/段落
- ❌ 模仿范文的开场方式（可以用同样的原理，不用同样的形式）
- ❌ 试图写出"像XXX一样的作品"

### 应该做的事
- ✅ 学习范文的**结构比例**（多长段落、多少对话、多久一个爽点）
- ✅ 学习范文的**信息投放节奏**（什么时候告诉读者什么）
- ✅ 学习范文的**情感曲线**（如何让读者从平静到紧张到释放）
- ✅ 学习范文的**人物塑造方法**（不只是性格标签，是行为模式）

### 转化模板

```
范文技法 → 本作品应用

技法：范文用"日常场景中的微小异常"制造悬念
应用：主角回家发现母亲做了他从不爱吃的菜，但她说"你小时候最爱吃这个"
差异：范文用的是环境异常（门开着），我用的是人物行为异常（记错喜好）
```

### 跨题材转化示例

```
灵异题材范文技法 → 都市题材转化

范文：用老旧照片上的陌生人建立悬念
转化：用手机相册里一张主角不记得拍过的照片建立悬念

范文：用"路会自己亮"来表现灵异能力
转化：用"手机自动弹出不该出现的导航路线"来表现都市异能
```

## 五、何时需要 / 何时跳过范文参照

### 建议使用范文参照的场景
- 初次创作网文，对节奏/结构没有把握
- 卡文了，需要看同类优秀作品找感觉
- 修订阶段发现数据指标明显低于行业标准
- 准备尝试新题材，不了解该题材的读者预期

### 建议跳过范文参照的场景
- 创意足够独特，强行对标会限制创造力
- 已有成熟的写作经验和稳定的读者群
- 写的是非常小众的题材，找不到合适的范文
- 修订后期，已经有明确的质量标准和修改方向

## 六、范文参照在流程中的位置

```
项目初始化 → 大纲设计 → 【可选：范文分析 → 技法提取】
                                       ↓
                                 应用到大纲/写作
                                       ↓
                             写作流程 → 三批次修订 → 终审

       跳过时：项目初始化 → 大纲设计 → 写作流程 → ...
```

在**大纲设计**完成后、**写作流程**开始前插入，是最佳时机。此时你已经有了自己的故事框架，范文不会"带偏"你的创意，只会给你的技法工具箱增加工具。

## 七、实操案例：《挑担》对标《大王饶命》

以下是我们在这部作品中的实际对标过程。

### 发现的问题（通过定量分析）

| 指标 | 《大王饶命》 | 《挑担》（修订前） | 差距 |
|------|------------|-----------------|------|
| 平均对话占比 | ~38% | ~22% | -16% |
| 章尾钩子覆盖率 | ~95% | ~50% | -45% |
| 平均每章字数 | ~2400 | ~2100 | -300 |
| 开篇钩子覆盖率 | 100% | ~60% | -40% |

### 采取的修订

1. **增加对话**：在叙述段落中插入冲突对话（A批次重点）
2. **补钩子**：逐章检查前后200字，补开场冲突和章尾悬念
3. **稳定字数**：将单章字数目标设定为2200-2600
4. **调整节奏**：缩短铺垫期，确保每3章内至少有一个小爽点

### 修订效果

| 指标 | 修订前 | 修订后 | 提升 |
|------|--------|--------|------|
| 对话占比 | ~22% | ~32% | +45% |
| 章尾钩子 | ~50% | ~90%+ | +80% |
| 开篇钩子 | ~60% | ~100% | +67% |
| 字数稳定度 | 波动大 | 2200-2600 | 稳定 |
